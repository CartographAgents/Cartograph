# Functional Requirements

## Purpose
Capture behavior the system must implement in testable terms.

## Inputs
- Product goals from `../00-context/business-goals.md`
- Architecture decisions from `../05-state/decisions-log.md`

## Outputs
- Prioritized functional requirement catalog
- Requirement-to-task traceability targets

## Required Sections
- Capability Inventory
- Requirement Statements (FR-###)
- Priority and Dependencies
- Acceptance Mapping

## Update Cadence
Update whenever requirements are added, removed, or re-prioritized.
