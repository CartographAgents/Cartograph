# Roadmap Mode Workflow

Use this mode to strategically steer the product or when the vision/roadmap diverges from the current technical backlog.

## Steps

1. **Vision-to-Roadmap Check**: Review `agent-pack/00-context/vision.md` and `agent-pack/00-context/roadmap.md` to ensure major milestones reflect the core mission.
2. **Current Backlog Audit**: Scan `agent-pack/04-task-system/` to identify missing features, skipped epics, or stale Roadmap items.
3. **Gap Identification**: Determine what features are missing from the task system that are required by the Roadmap Phase.
4. **Roadmap Refinement**: If necessary, update milestones or phases in `roadmap.md` based on technical feasibility or user feedback.
5. **Backlog Population (Transition to Task Mode)**: Create or update Epics and Features to bridge discovered gaps.
6. **Log Strategic Decisions**: Record the rationale for any major Roadmap or Vision shift in `agent-pack/05-state/decisions-log.md`.
7. **Handoff Documentation**: Update `agent-pack/05-state/handoff-notes.md` with strategic priorities for the next agent cycle.
